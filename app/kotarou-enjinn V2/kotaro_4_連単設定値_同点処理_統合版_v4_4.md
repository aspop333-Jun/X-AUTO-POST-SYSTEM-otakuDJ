# kotaro_4_連単設定値_同点処理_統合版（V4.4 / Anti-P02 Lock）
**目的**：Pose Patch導入後に発生した **P02（余韻/Perform）82%ロック**を解除し、  
「無難ポーズ（体斜め・顔正面）」は **“演出”ではなく“無難”**として扱う（＝P01へ戻す）。  
さらに、P01/P02 の分岐を **C>D の単純比較から、条件トリガー式**へ変更して再現性を上げる。

---

## 0. 事実（あなたのV4.1 Poseベンチから）
- 多数で `pose_safe_theory=1`（体斜め・顔正面）が立つ
- その結果 `C += 0.6` が効きすぎて、A主役でも `C > D` が頻発
- 旧A分岐（`D>=C→P01 / C>D→P02`）により **P02へ吸い込み**

---

## 1. Pose補正の再設計（効き過ぎ防止）
### 1.1 pose_safe_theory（体斜め・顔正面）は「無難」であり、Performではない
- `pose_safe_theory=1` → **C += 0.2**（旧 +0.6 から弱める）
- `pose_safe_theory=1` → **E -= 0.4**（距離感は維持）

> ※「無難＝クール演出」ではないため C加点を下げる  
> ※距離（E）は下げる（あなたの現場感覚どおり）

### 1.2 pose_side_cool（体斜め・顔も斜め）＝鑑賞距離/クール
- `pose_side_cool=1` → **C += 0.7 / E -= 0.5**（据え置き）

### 1.3 pose_front_true（真正面）＝親密
- `pose_front_true=1` → **E += 0.6 / D += 0.2 / A += 0.2**（据え置き）

### 1.4 所作優先（あなたの条件）
- `talk_to=1` または `casual_moment=1` の場合、  
  Pose由来の **E減点は半分**（-0.2）にする

---

## 2. 二次加点（V4.3踏襲）
- `casual_moment` → `A += 0.7`（+0.2追加ブーストは維持）
- `nostalgic` → `A += 0.5`
- `crowd_venue` → `B += 0.7`
- `group_feeling` → `B += 0.5` + `C += 0.5`
- `talk_to` → `D += 0.5`（+0.2追加ブースト維持）
- `close_dist` → `D += 0.3`
- `costume_strong` → `C += 0.7`
- `act_point_or_salute` → `C += 0.5` + `D += 0.3`
- `prop_strong` → `B += 0.7`

クリップ：A,B,C,D,E は 0〜5、二次加点合計は +1.5 以内。

---

## 3. A主役（P01/P02）分岐の変更（最重要）
### 3.1 旧（禁止）
- ❌ `D>=C → P01` / `C>D → P02`
> 固定スコア環境だと、微差でロックする

### 3.2 新（トリガー式）
A主役のとき：

- **P02（余韻/Perform）**にする条件：  
  - `costume_strong=1` または  
  - `act_point_or_salute=1` または  
  - `pose_side_cool=1` または  
  - `C - D >= 0.8`（明確に演出が勝つ）

- 上記に当てはまらない → **P01（余韻/Soft）**

### 3.3 強制セーフ（無難ポーズはSoftへ）
- `pose_safe_theory=1` **かつ** `costume_strong=0` **かつ** `act_point_or_salute=0` **かつ** `pose_side_cool=0`  
  → **強制で P01**

> あなたの現場感覚どおり：  
> 「体斜め・顔正面」は無難で綺麗だが“面白みが薄い”＝Performではない

---

## 4. B主役（P03/P04）分岐（V4.3維持）
B主役のとき：
- `crowd_venue=1` または `group_feeling=1` → **P03**
- それ以外で `prop_strong=1` → **P04**
- それ以外で `B - A <= 0.5` → **P03**
- それ以外 → **P04**

---

## 5. フラット逃がし（V4.3維持）
- `M = max(A,B,C,D)`
- `M <= 2` → P11/P12（状況強→P12、そうでなければP11）

---

## 6. ログ仕様（必須）
- `main`: A/B/C/D/None
- `sub4`: 例 `A>B>C>D`
- `pattern_id`: P01〜P12
- `flags`: すべて
