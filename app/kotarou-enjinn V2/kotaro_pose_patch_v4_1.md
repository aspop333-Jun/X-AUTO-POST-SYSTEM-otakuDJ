# Kotaro Scoring V4.x Pose Patch（体の角度＝親密性/無難さの補助判定）
**目的**：あなたの現場感覚を「定量フラグ」に落として、E（親近感）とC（クール/演出）の分布を安定させる。  
**要点**：ポーズには二系統ある  
- **無難で綺麗（セオリー）**：体は斜め／顔だけ正面 → かっこよく見えるが、親密性は相対的に低い  
- **親密性優先**：真正面（特に真正面） → 映りは度外視になりがちだが、距離が近く出やすい

---

## 1) 新フラグ（0/1）
VLMで画像から判定して、既存の Flags に追加する。

- `pose_safe_theory`：**体=斜め + 顔=正面**（無難・綺麗・機械的になりやすい）
- `pose_front_true`：**体=正面 + 顔=正面（真正面）**（親密寄り）
- `pose_side_cool`：**体=斜め + 顔=斜め（横向き寄り）**（鑑賞距離/クール寄り）
- `pose_front_body_face_angled`：体=正面 + 顔=斜め（例外ケース）

> 実装的には「肩の向き（胴体）」「顔の向き（目鼻の向き）」の2軸で判定。

---

## 2) 判定質問（VLMに投げる用：Pose Q）
※このまま質問セットに入れてOK（0/1）

- PZ01: 肩（胴体）がカメラに対して斜め（3/4）になっている
- PZ02: 肩（胴体）が真正面に近い
- PZ03: 顔（目鼻）が正面を向いている
- PZ04: 顔が斜め〜横向き寄り（目鼻の角度がついている）
- PZ05: 「体は斜め・顔だけ正面」になっている（=pose_safe_theory）
- PZ06: 「体も顔も真正面」に近い（=pose_front_true）

## 3) V4 二次加点に追加（分布散らし・人格は奪わない）

（合計+1.5クリップは維持）

### 3.1 推奨加点（固定）
- `pose_safe_theory=1` → `C += 0.6` / `E -= 0.4`
- `pose_side_cool=1` → `C += 0.7` / `E -= 0.5`
- `pose_front_true=1` → `E += 0.6` / `D += 0.2` / `A += 0.2`
- `pose_front_body_face_angled=1` → `A += 0.3` / `B += 0.2`

> 意図：セオリー系を「距離があるクール」に寄せ、真正面を「親密」に寄せる。  
> ただし `talk_to` 等の“所作フラグ”が立つ場合は、そちらを優先（下記）。

### 3.2 所作優先（あなたの条件をルール化）
- `talk_to=1` または `casual_moment=1` が立っている場合、  
  Pose由来の **E減点は半分**（-0.2 / -0.25 など）にする  
  → 「親近感のある所作があるならそっち優先」を実装する
