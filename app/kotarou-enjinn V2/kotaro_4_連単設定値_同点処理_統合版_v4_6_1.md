# kotaro_4_連単設定値_同点処理_統合版（V4.6.1 / De-Cluster P01 → P03 “効きすぎ修正”）
**目的**：V4.6で P03=82% に振り切れたため、P01→P03 の逃がし条件を「弱い親密」だけに限定する。  
結論：`A - B` 閾値を絞るより、**“Bが強い個体だけ逃がす”** ほうが安定する（あなたのセットは A-B=0.5 が多数で、一律閾値だと全吸い込みになる）。

---

## 変更点（ここだけ差し替え）
V4.6 の「3.1 P03へ逃がす条件」を、下記に置換する。

### 3.1 P03へ逃がす条件（V4.6.1）
**前提**：このブロックは「A主役でP01候補」になった個体にだけ適用する  
（B主役/他主役には絶対に適用しない）

- `main == "A"` かつ `pattern_candidate == "P01"`
- `pose_safe_theory == 1`
- `talk_to == 1`
- `casual_moment == 1`
- `intimacy_strong == 0`
- `crowd_venue == 0`
- `group_feeling == 0`
- `costume_strong == 0`
- `act_point_or_salute == 0`
- **追加ゲート（V4.6.1）**：`B >= 4.2` かつ `A - B <= 0.6`

→ すべて満たすと **pattern = P03**（構図/Scene）へ振り替える

---

## なぜこれが効くか（今回ログに対して）
あなたのV4.6ログで大量に P03 へ流れた個体は、典型的に

- `A=3.9, B=3.4`（A-B=0.5）  
- `pose_safe_theory + talk_to + casual_moment`

この「A-B=0.5帯」が母集団の大半なので、`A-B<=0.6` だと **ほぼ全件ヒット**する。  
V4.6.1は **Bが本当に強い（>=4.2）** ときだけ “構図寄り” と判断し、P03へ逃がす。

---

## ノブ（1回で収束）
- 逃がしを増やしたい → `B >= 4.0` に下げる  
- 逃がしを減らしたい → `B >= 4.4` に上げる  

※ `A - B` は 0.6 のままでOK（今回の問題はここじゃなく「母集団が全部0.5」）
